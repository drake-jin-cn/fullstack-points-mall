### Background

All 8 services currently have hardcoded port numbers and no documented environment variable contracts.
There are no `.env.example` files (except Laravel's minimal one), no multi-environment profiles
(dev/test/prod), and environment variable loading is not configured at startup. This blocks
environment parity and makes secrets management risky.

### Goals

1. Every service has a `.env.example` documenting all required variables with safe placeholder values.
2. Every service has `dev`, `test`, and `prod` environment profiles (framework-appropriate format).
3. Every service loads env vars at startup from the correct profile file (no hardcoded secrets).
4. No secrets (passwords, API keys, tokens) appear in any source file committed to Git.
5. All real `.env.*` files (except `.env.example`) are in `.gitignore`.

### Out of Scope

- Actual secret values (provided by operators, never committed)
- Docker Compose `.env.docker` override (deferred to T090)
- Kubernetes secret management (deferred to T089)
- Runtime secret rotation

### Technical Design

#### Environment profile strategy per framework

| Service | Framework | Profile Mechanism |
|---------|-----------|-------------------|
| `bff` | NestJS | `@nestjs/config` â†’ loads `.env.${NODE_ENV}` |
| `core` | Spring Boot | Spring profiles â†’ `application-{profile}.yml` |
| `shop` | Laravel | `.env` file (single, operator-managed) + documented profiles |
| `message` | Express/TS | `dotenv` â†’ loads `.env.${NODE_ENV}` |
| `thirdparty` | Spring Boot WebFlux | Spring profiles â†’ `application-{profile}.yml` |
| `data` | FastAPI | `python-dotenv` â†’ loads `.env.${ENVIRONMENT}` |
| `frontend` | Next.js | Built-in Next.js env â†’ `.env.{development,test,production}` |
| `frontend-base` | Rollup | `.env.example` only (build-time, no runtime) |

#### Variable namespacing

- All secrets use empty placeholder `=` (never a real value in `.env.example`)
- Non-secret defaults use sensible localhost values
- Variables requiring operator action are commented with `# Required`
