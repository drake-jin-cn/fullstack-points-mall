### Background

All Phase 1+ feature tasks (auth, points, shop, attendance) depend on the database tables being
created and reachable. Without an authoritative schema, any service that starts will either crash
(DDL missing) or deviate from the agreed contract. We need a single source of truth before any
service writes a single `SELECT`.

### Goals

1. Define the full DDL for **`points_core`** PostgreSQL database (6 tables).
2. Define the full DDL for **`points_shop`** PostgreSQL database (6 tables).
3. Deliver the schema as **versioned migration files** so the DB evolves safely.
4. Document human-readable **ER diagrams** in `.wiki/db/` as the canonical schema reference.
5. Verify both databases can be created and migrated successfully via `docker-compose`.

### Out of Scope

- No seed / fixture data (that belongs to T006 employee seeder).
- No ORM entity classes (those are generated in T006 / T034).
- No full-text search indexes or partitioning (deferred to Phase 7).
- No Redis key schema (addressed in separate INFRA tasks).
- No `points-mall-message`, `points-mall-data`, or `points-mall-thirdparty-connector` DBs
  (those services are stateless or use the core DBs via APIs).

### Technical Design

#### docker-compose additions

Add a `docker-compose.yml` at project root with three services:

```yaml
services:
  postgres:
    image: postgres:16-alpine
    environment:
      POSTGRES_USER: points
      POSTGRES_PASSWORD: points_dev
    ports: ["5432:5432"]
    volumes:
      - pgdata:/var/lib/postgresql/data
      - ./infra/postgres/init.sql:/docker-entrypoint-initdb.d/init.sql

  redis:
    image: redis:7-alpine
    ports: ["6379:6379"]

  rabbitmq:
    image: rabbitmq:3-management-alpine
    ports:
      - "5672:5672"
      - "15672:15672"

volumes:
  pgdata:
```

`infra/postgres/init.sql` creates both databases:
```sql
CREATE DATABASE points_core;
CREATE DATABASE points_shop;
```

#### points_core — Flyway migrations

Location: `points-mall-core/src/main/resources/db/migration/`  
Naming: `V{n}__{snake_case_description}.sql`

**V1__create_departments.sql**
```sql
CREATE TABLE departments (
    id          BIGSERIAL PRIMARY KEY,
    name        VARCHAR(100) NOT NULL,
    code        VARCHAR(50)  NOT NULL UNIQUE,
    description TEXT,
    parent_id   BIGINT REFERENCES departments(id) ON DELETE SET NULL,
    created_at  TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);
```

**V2__create_employees.sql**
```sql
CREATE TABLE employees (
    id            BIGSERIAL    PRIMARY KEY,
    name          VARCHAR(100) NOT NULL,
    email         VARCHAR(255) NOT NULL UNIQUE,
    password_hash VARCHAR(255),                   -- nullable: OAuth-only users have no password
    department_id BIGINT REFERENCES departments(id) ON DELETE SET NULL,
    github_id     VARCHAR(100) UNIQUE,
    avatar_url    TEXT,
    is_active     BOOLEAN      NOT NULL DEFAULT TRUE,
    created_at    TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    updated_at    TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_employees_email ON employees(email);
CREATE INDEX idx_employees_department_id ON employees(department_id);
```

**V3__create_roles.sql**
```sql
CREATE TABLE roles (
    id          BIGSERIAL   PRIMARY KEY,
    name        VARCHAR(50) NOT NULL UNIQUE,  -- e.g. 'admin', 'employee', 'manager'
    description TEXT,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE TABLE employee_roles (
    employee_id BIGINT NOT NULL REFERENCES employees(id)  ON DELETE CASCADE,
    role_id     BIGINT NOT NULL REFERENCES roles(id)      ON DELETE CASCADE,
    granted_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (employee_id, role_id)
);

INSERT INTO roles (name, description) VALUES
    ('admin',    'Full system administrator'),
    ('employee', 'Regular employee');
```

**V4__create_attendance_records.sql**
```sql
CREATE TABLE attendance_records (
    id            BIGSERIAL   PRIMARY KEY,
    employee_id   BIGINT      NOT NULL REFERENCES employees(id) ON DELETE CASCADE,
    check_in_at   TIMESTAMPTZ NOT NULL,
    check_out_at  TIMESTAMPTZ,
    work_date     DATE        NOT NULL,
    status        VARCHAR(20) NOT NULL DEFAULT 'normal',  -- normal | late | early_leave | absent
    notes         TEXT,
    created_at    TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at    TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_attendance_employee_date ON attendance_records(employee_id, work_date);
```

**V5__create_points_rules.sql**
```sql
CREATE TABLE points_rules (
    id            BIGSERIAL    PRIMARY KEY,
    rule_code     VARCHAR(50)  NOT NULL UNIQUE,  -- e.g. 'CHECKIN_DAILY', 'ATTENDANCE_FULL_MONTH'
    description   TEXT         NOT NULL,
    points_value  INT          NOT NULL,         -- positive = earn, negative = deduct
    is_active     BOOLEAN      NOT NULL DEFAULT TRUE,
    valid_from    DATE,
    valid_until   DATE,
    created_at    TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    updated_at    TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);
```

**V6__create_points_ledger.sql**
```sql
CREATE TABLE points_ledger (
    id            BIGSERIAL    PRIMARY KEY,
    employee_id   BIGINT       NOT NULL REFERENCES employees(id) ON DELETE CASCADE,
    rule_id       BIGINT       REFERENCES points_rules(id) ON DELETE SET NULL,
    delta         INT          NOT NULL,         -- positive = earn, negative = spend
    balance_after INT          NOT NULL,
    description   TEXT,
    ref_type      VARCHAR(50),                   -- 'attendance' | 'order' | 'manual' | 'system'
    ref_id        BIGINT,                        -- FK to the triggering entity (nullable)
    created_at    TIMESTAMPTZ  NOT NULL DEFAULT NOW()
);
CREATE INDEX idx_ledger_employee_id ON points_ledger(employee_id);
CREATE INDEX idx_ledger_created_at  ON points_ledger(created_at);
```

Flyway config in `application-dev.yml` / `application-prod.yml`:
```yaml
spring:
  flyway:
    enabled: true
    locations: classpath:db/migration
    baseline-on-migrate: false
```

#### points_shop — Laravel migrations

Location: `points-mall-shop/database/migrations/`  
Each file is a standard Laravel migration class.

| # | File | Table |
|---|------|-------|
| 1 | `2026_06_23_000001_create_categories_table.php` | `categories` |
| 2 | `2026_06_23_000002_create_products_table.php` | `products` |
| 3 | `2026_06_23_000003_create_orders_table.php` | `orders` |
| 4 | `2026_06_23_000004_create_order_items_table.php` | `order_items` |
| 5 | `2026_06_23_000005_create_menu_items_table.php` | `menu_items` |
| 6 | `2026_06_23_000006_create_announcements_table.php` | `announcements` |
| 7 | `2026_06_23_000007_create_system_configs_table.php` | `system_configs` |

Key columns:

- **categories**: `id`, `name`, `slug` (unique), `parent_id` (self-ref), `sort_order`, `is_active`, timestamps
- **products**: `id`, `category_id` (FK), `name`, `slug` (unique), `description`, `image_url`, `points_cost` (unsignedInt), `stock` (unsignedInt), `is_active`, timestamps
- **orders**: `id`, `employee_id` (int, not FK — cross-DB reference), `total_points`, `status` (enum: pending/paid/shipped/completed/cancelled), timestamps
- **order_items**: `id`, `order_id` (FK), `product_id` (FK), `quantity`, `points_cost_each`, timestamps
- **menu_items**: `id`, `title`, `url`, `icon`, `parent_id` (self-ref), `sort_order`, `required_role` (nullable), `is_active`, timestamps
- **announcements**: `id`, `title`, `content` (text), `is_pinned`, `published_at`, `expires_at`, `created_by` (employee_id), timestamps
- **system_configs**: `id`, `key` (string, unique), `value` (text), `description`, timestamps
