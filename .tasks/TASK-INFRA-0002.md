## Raw Requirements

`[root]` Build the project-level toolchain so that the WORKFLOW.md Step 10–11 commands actually work:
`pnpm run tasks:sync` rebuilds `.tasks/_index.md`; `pnpm run test:task TASK-XXX --update-status`

### Background

TASK-INFRA-0001 is `dev-done` and all 8 services pass `GET /health → 200 OK` manually. However,
WORKFLOW.md Step 10 (`pnpm run tasks:sync`) and Step 11 (`pnpm run test:task ... --update-status`)
cannot run because:

1. There is no root-level `package.json` — `pnpm` has nowhere to define workspace scripts
2. `.tests/scripts/` does not exist — the three scripts are not yet written
3. Bruno CLI is not installed — Bruno `.bru` test files exist but cannot be executed
4. No commit-msg hook exists — developers can commit with any message format, making
   task traceability in `git log` unreliable. Each submodule is an independent git repo,
   so the root Husky config does NOT cover them automatically.

This task creates the minimum viable toolchain to close the WORKFLOW loop end-to-end.

### Goals

1. Root `package.json` with `pnpm` workspace config and the following scripts:
   - `tasks:list [--status=<status>]` — print task overview from `_index.md`
   - `tasks:sync` — rebuild `.tasks/_index.md` from all task files
   - `tasks:view <TASK-ID>` — print a single task file
   - `test:task <TASK-ID> [--update-status]` — run tests for a task, optionally update status
   - `check:spec-api` — placeholder (exits 0 for now; real implementation in a later task)
2. `pnpm-workspace.yaml` listing all service directories
3. `.tests/scripts/tasks-sync.js` — scans all `.tasks/TASK-*.md`, extracts YAML frontmatter,
   writes a markdown summary table to `.tasks/_index.md`
4. `.tests/scripts/run-task-tests.js` — reads `test_refs` from the specified task file,
   dispatches test types:
   - `*.bru` files → `bru run <file> --env local`
   - `*.test.ts` files → `vitest run <file>`
   - `*.spec.ts` files → `playwright test <file>`
   Aggregates pass/fail; if `--update-status` is set, rewrites task status and appends
   history record
5. Bruno CLI installed as a dev dependency (`@usebruno/cli`)
6. `.tests/api/README.md` — documents Bruno environment variables and how to run tests
7. Husky installed; `commit-msg` hook enforces format `<type>(TASK-ID): <summary>` on root repo
8. `.husky/commit-msg` script shared via `.git-hooks/commit-msg` — a single hook script
   that lives in version control and is symlinked/copied into each repo
9. `hooks:install` script — installs the hook into root + all 8 submodule `.git/hooks/` directories
   so any developer who clones with submodules can run `pnpm run hooks:install` once to be covered

### Technical Design

#### Root package.json structure

```json
{
  "name": "fullstack-points-mall",
  "private": true,
  "scripts": {
    "tasks:list":   "node .tests/scripts/tasks-sync.js --list",
    "tasks:sync":   "node .tests/scripts/tasks-sync.js",
    "tasks:view":   "node .tests/scripts/tasks-sync.js --view",
    "test:task":    "node .tests/scripts/run-task-tests.js",
    "check:spec-api": "echo 'check:spec-api not yet implemented' && exit 0",
    "hooks:install": "node .git-hooks/install.js",
    "prepare": "node .git-hooks/install.js"
  },
  "devDependencies": {
    "@usebruno/cli": "^2.0.0",
    "husky": "^9.0.0"
  }
}
```

#### pnpm-workspace.yaml

```yaml
packages:
  - 'points-mall-*'
```

#### tasks-sync.js responsibilities

- Parse YAML frontmatter from every `.tasks/TASK-*.md`
- Generate `_index.md` with a status-grouped table: id / title / status / assignee / updated
- Support `--list [--status=X]` (print to stdout, no file write)
- Support `--view TASK-ID` (print that task file to stdout)

#### run-task-tests.js responsibilities

```
1. Read task ID from argv (e.g. TASK-INFRA-0001)
2. Parse the task file's test_refs frontmatter field
3. For each test_ref:
   - *.bru  → run: bru run <path> --env local
   - *.test.ts → run: pnpm --filter <service> test <path>  (future)
   - *.spec.ts → run: playwright test <path>  (future)
4. Collect exit codes
5. If --update-status:
   - all pass → rewrite status: test-pass, append history
   - any fail → rewrite status: test-fail, append failure summary
6. Exit with 0 (all pass) or 1 (any fail)
```

#### Git hook design

All repos (root + 8 submodules) enforce the same format:

```
<type>(<scope>): <summary>

where:
  type  = feat | fix | chore | test | docs | refactor | ci
  scope = TASK-ID (e.g. TASK-INFRA-0001) or service name
```

Implementation:

```
.git-hooks/
  commit-msg        # the actual shell hook script (version-controlled)
  install.js        # Node.js installer — copies hook into root + all submodule .git/hooks/
```

`install.js` logic:
```
1. Determine root repo .git/hooks/ path
2. Read submodule paths from .gitmodules (parse [submodule "..."] path = ... entries)
3. For each repo (root + submodules):
   - Create hooks/ dir if missing
   - Copy .git-hooks/commit-msg → <repo>/.git/hooks/commit-msg
   - chmod +x
4. Print success summary
```

Note: Using copy (not symlink) so the hook works correctly when checked out on both
macOS and Windows developer machines.

`prepare` script runs `hooks:install` automatically after `pnpm install`, so new team
members get hooks without a manual step.

#### Bruno environment config

`.tests/api/bruno.json` already exists with `local` environment defining per-service base URLs.
`bru run` needs the `--env local` flag. Scripts pass this automatically.
