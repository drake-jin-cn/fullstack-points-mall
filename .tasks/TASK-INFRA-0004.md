### Background

TASK-INFRA-0001 established a minimal `GET /health â†’ { status: "ok" }` skeleton on all 8 services.
TASK-INFRA-0003 wired multi-environment config so each service reads `PORT`, `DB_*`, `REDIS_*` from env.
Now the health endpoint needs to evolve into a real liveness + readiness signal: it must report
whether the DB connection is reachable, how long the process has been running, and which service it
belongs to â€” so operators and the future `/admin/system/health` dashboard page (T094) can tell at a
glance whether each service is healthy.

`frontend` and `frontend-base` are excluded: Next.js already exposes a framework-managed health
route, and the base library has no runtime server.

### Goals

1. All 6 backend services (`bff`, `core`, `shop`, `message`, `thirdparty-connector`, `data`) return
   the standardized payload from `GET /health`.
2. The `db` field reflects the actual DB connectivity state: `"ok"` if a lightweight probe succeeds,
   `"error"` if it fails or is not yet configured.
3. The `uptime` field reports process uptime in seconds (integer).
4. The `timestamp` field is the current UTC ISO 8601 datetime string.
5. The `service` field is a human-readable service name string.
6. The endpoint always returns HTTP 200 (health check must not crash the process on DB error).
7. No secrets (DB passwords, etc.) appear in the response.

### Out of Scope

- `redis` and `mq` connectivity fields â€” deferred to T093 (full readiness probe in Phase 10)
- Actual DB tables / schema â€” deferred to T003; DB probe uses a lightweight connection check only
  (e.g. `SELECT 1` for SQL, no table scan)
- HTTP 503 on degraded state â€” deferred to T093
- Metrics / tracing integration (Prometheus, OpenTelemetry) â€” out of project scope

### Response Contract

```json
{
  "status": "ok",
  "service": "points-mall-bff",
  "timestamp": "2026-06-23T10:00:00.000Z",
  "db": "ok",
  "uptime": 42
}
```

`db` is `"ok"` when a connection probe succeeds, `"error"` when it fails or when no DB is configured
for this service (e.g. `bff` has no direct DB â€” returns `"ok"` by convention since no DB = no DB
failure). The `status` field is always `"ok"` at this phase (T093 will add `"degraded"`).

### Service-by-Service Design

| Service | Framework | DB probe | `db` when no DB env set |
|---------|-----------|----------|-------------------------|
| `bff` | NestJS | No direct DB â€” always `"ok"` | `"ok"` |
| `core` | Spring Boot | `SELECT 1` via DataSource | `"error"` (DB vars required) |
| `shop` | Laravel | `DB::select('SELECT 1')` | `"error"` |
| `message` | Express/TS | No direct DB yet â€” always `"ok"` | `"ok"` |
| `thirdparty-connector` | Spring Boot WebFlux | No direct DB â€” always `"ok"` | `"ok"` |
| `data` | FastAPI | No direct DB yet â€” always `"ok"` | `"ok"` |

> Core and Shop are the only services with a DB connection at this phase. All others will get real
> DB probes when their DB is set up in T003/T006.

#### BFF (NestJS)

- Update `HealthController.check()` to return the full payload.
- Use `process.uptime()` (seconds, rounded to integer) for `uptime`.
- `db: "ok"` always (no direct DB).

#### Core (Spring Boot)

- Inject `DataSource`; in health method try `dataSource.getConnection().isValid(1)`.
  Catch any exception â†’ `db: "error"`.
- `ManagementFactory.getRuntimeMXBean().getUptime() / 1000` (integer seconds).
- `Instant.now().toString()` for timestamp.

#### Shop (Laravel)

- In `HealthController` try `DB::select('SELECT 1')` wrapped in try/catch â†’ `db: "ok"|"error"`.
- `round(microtime(true) - LARAVEL_START)` for uptime (define `LARAVEL_START` in `public/index.php`
  if not already defined by framework â€” Laravel 11 defines it automatically).

#### Message (Express/TS)

- Track `const startTime = Date.now()` at module load; `uptime = Math.floor((Date.now() - startTime) / 1000)`.
- `db: "ok"` always (no DB at this phase).

#### ThirdParty Connector (Spring Boot WebFlux)

- Return `Mono<Map<String, Object>>` with full payload.
- `db: "ok"` always (no DB).
- Use `ManagementFactory.getRuntimeMXBean().getUptime() / 1000`.

#### Data (FastAPI)

- `import time; uptime = int(time.time() - _start_time)` where `_start_time = time.time()` set at module load.
- `db: "ok"` always (no DB at this phase).
- `datetime.utcnow().isoformat() + "Z"` for timestamp.
