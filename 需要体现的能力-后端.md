1. 后端技术选型：基于Next.js API Route / NestJS 轻量后端，统一Node.js+TS全栈技术栈，前后端技术栈统一无割裂
2. RESTful API 规范落地：遵循标准REST接口设计，区分GET/POST/PUT/DELETE请求，使用行业标准HTTP状态码，接口语义清晰统一
3. 后端分层架构设计：采用轻量化MVC分层，拆分控制器层、服务层、数据模型层，解耦接口接收、业务逻辑、数据库操作，代码结构清晰可维护
2.2 数据库设计与ORM操作（必做，全栈基础）
1. 数据库选型与表结构设计：选用PostgreSQL关系型数据库，完成用户、菜单、权限、业务表单核心表设计，梳理表之间一对多、多对多基础关联关系
2. ORM框架选型：采用Prisma做类型安全ORM，全链路TS类型约束，实现前后端接口、数据库表结构类型统一，杜绝类型不一致问题
3. 数据库基础优化：给高频查询字段建立索引，规避全表扫描等低级低效SQL；理解数据库事务，完成登录、数据新增等场景简单事务兜底
2.3 认证鉴权与后端安全（高分核心，和前端安全联动）
1. 登录认证体系：基于JWT实现无状态登录鉴权，密码采用bcrypt加盐哈希加密，数据库不存储明文密码
2. 分层权限控制：实现管理员/普通用户两层基础权限，对接前端三维权限体系，后端做接口权限二次校验，避免前端权限绕过漏洞
3. 后端基础安全防护：后端接口层面防范XSS注入、CSRF跨站请求伪造，请求参数统一校验过滤，补齐前端安全防护盲区
2.4 第三方平台API集成（后端核心加分项，海外面试高频考点）
设计初衷：第三方接口对接鉴权逻辑繁杂、各平台签名规则、响应格式差异极大，如果让每个业务服务直接对接外部三方，会造成代码耦合严重、维护成本高。因此单独抽离ThirdPartyConnector第三方聚合服务，全站统一收口所有外部请求，业务服务只需要调用内部标准化接口，无需感知外部第三方底层差异，是后端工程化解耦的经典实践。
整体对接原则：仅做标准REST/OAuth接口对接，封装统一第三方请求工具类，做好超时重试、异常捕获、参数适配；不深入第三方底层源码，守住前端主技术栈定位，不过度后端化。
1. BFF(NestJS) → GitHub OAuth 第三方登录业务场景：员工免密快捷登录系统，海外企业内部系统主流登录方案
2. 对接能力：OAuth2.0授权流程、令牌换取、用户信息解析、本地账号绑定、登录状态同步
3. ThirdPartyConnector → AWS S3 亚马逊云对象存储通俗释义：AWS出品的海外标准云端对象存储，对标阿里云OSS / Cloudflare R2，专门用于服务器文件云端托管
4. 业务场景：存放员工考勤归档文件、月度敏感报表离线备份、系统长期静态文件归档
5. 对接能力：调用AWS官方SDK，生成安全预签名URL、配置防盗链权限、文件冷热分层存储，保障后端敏感归档文件安全
6. 场景区分：前端日常商品图片用R2，后端归档备份文件用S3，贴合企业真实云存储分层使用规范
7. 对接能力：AWS官方SDK调用、文件预签名URL生成、权限防盗链配置、大文件云端备份
8. PHP Laravel服务 → Amazon Affiliate 亚马逊商品数据源业务场景：内部积分商城同步亚马逊公开好物作为兑换商品，丰富商城货源，贴合海外生态
9. 对接能力：亚马逊公开API签名鉴权、第三方数据格式化、适配本地商城数据表结构、定时增量同步商品数据
10. Node服务 → SendGrid 海外邮件推送服务业务场景：员工生日积分发放、考勤异常提醒、订单兑换结果、账号安全预警邮件推送
11. 对接能力：模板邮件定制、异步邮件发送、发送失败重试、发送日志记录
通用第三方对接话术（直接背）：我在项目中为四个异构后端服务分别接入海外主流第三方平台，完整覆盖OAuth授权、云存储、外部商品数据源、消息推送四类后端常见对接场景；统一封装第三方请求拦截器，处理不同平台的鉴权规则、响应格式差异与网络异常，具备完整的第三方接口联调与容错处理能力。
30秒专项口述话术：为什么单独抽离ThirdPartyConnector（满分标准答案）：我之所以单独抽离第三方聚合连接器，核心有三点考量。第一是解耦业务与外部依赖，各个业务服务只负责自身核心业务，不用接入五花八门的第三方SDK、适配差异化签名规则，避免业务代码臃肿；第二是统一治理外部请求，把超时重试、统一降级、密钥管理、响应格式归一化全部收拢在一处，全局第三方请求规范统一；第三是提升可维护性，后续新增或者更换第三方服务商，只需要修改连接器内部代码，完全不用改动上游业务服务和BFF网关，符合开闭原则，同时也规避了外部不稳定三方接口直接影响核心业务的风险。
2.5 后端工程化与接口规范（体现后端专业度）
1. 请求参数统一校验：后端复用Zod做参数强校验，前后端校验规则一致，非法参数直接拦截，减轻数据库查询压力
2. 全局统一响应与错误处理：封装后端全局异常拦截器，统一接口返回格式、统一错误码，和前端响应拦截器完美适配
3. 后端结构化日志：记录接口请求日志、错误日志、耗时日志，方便线上问题快速排查
4. 自动API文档生成：基于OpenAPI+Swagger自动生成在线接口文档，接口入参、出参、错误码一目了然，提升团队协作效率
2.6 多环境配置与线上部署（海外面试极度看重）
1. 多环境隔离配置：区分开发/测试/生产三套后端环境变量，隔离数据库地址、密钥、第三方SDK密钥，密钥不硬编码在代码中
2. Docker容器化部署：编写后端专属Dockerfile，前后端统一容器化部署，部署流程标准化
3. 线上可演示能力：后端服务、数据库、第三方能力全部线上可访问，完整支持Demo全流程演示，无本地才能运行的限制
