React+Next.js(App Router) Demo 全量开发清单（可直接照着逐点开发）
基础信息
技术栈：React18 + Next.js(App Router) + TypeScript + TailwindCSS + Zustand + React Query + Zod
状态库选型设计思路（仅思路，无代码）：针对Next.js App Router服务端组件RSC架构，横向对比三款主流React状态管理库，最终敲定Zustand：1. Recoil强依赖顶层Provider包裹，服务端水合报错严重，生态更新停滞，完全不适配Next；2. Jotai偏向原子化细粒度状态，适合小组件共享状态，不利于B端后台用户权限、全局主题等大块模块化全局状态维护；3. Zustand零Provider侵入、天然兼容RSC无水合问题，支持状态模块化拆分，自带持久化中间件，完美适配本次企业级后台全局状态管理场景，因此放弃另外两款库。
项目定位：全栈偏前端 SaaS后台Demo，侧重前端工程化、Next原生能力、线上项目稳定性，不深耕重型后端底层能力，适配海外前端/全栈岗面试
开发规则：先做完【已掌握必做项】夯实基础，再做【进阶核心必做项】拉高竞争力，最后补【低成本加分项】，全程避开后端冗余功能

4. API 接口降级策略
  - 接口报错/网络异常时，自动返回兜底默认静态数据

Part2：核心进阶能力开发（简历核心亮点，必做）
2.1 Next.js 原生高阶能力（Next专属核心加分）
1. RSC服务端/客户端组件精准划分
  - 规范使用'use client'，杜绝客户端组件滥用
  - 敏感密钥、接口请求放在服务端组件，避免前端密钥泄露
  - 减少客户端水合压力，优化页面初始渲染性能
2. 多渲染策略落地：SSG / ISR / SSR
  - 静态营销页：SSG静态预渲染，极致首屏速度
  - 动态内容页：ISR增量静态再生，定时刷新缓存，平衡性能与实时性
  - 个人中心/权限页：SSR服务端动态渲染，保证登录态、权限实时校验
3. Next高阶路由能力
  - 平行路由+拦截路由：无页面跳转，实现全局弹窗、详情浮层
  - 路由组：模块化拆分页面，不污染浏览器地址栏URL
  - 原生兜底页面：loading.tsx全局加载、error.tsx页面级报错、not-found.tsx404兜底
4. SEO全量升级（补强你原有基础SEO）
  - Next Metadata静态+动态元数据自动生成
  - OG社交分享标签、推特卡片适配（海外岗位刚需）
  - 自动生成sitemap.xml、robots.txt爬虫配置文件
  - JSON-LD结构化数据，提升搜索引擎收录权重
2.2 请求层高阶增强（基于拦截器/异常/降级做升级）
1. React Query 服务端状态管理
  - 接口自动缓存、窗口聚焦自动重请求、缓存过期失效
  - 统一接管请求loading/error状态，清理组件冗余状态代码
  - 乐观更新UI，无需等待接口返回，提升交互流畅度
  - 支持接口串行/并行请求、定时轮询
2. 请求层生产级优化
  - 重复请求自动取消，避免重复发包
  - 接口超时自动重试，搭配原有API降级，双层故障兜底
  - 无感Token刷新：401状态码自动续期令牌，用户无感知退出登录
2.3 鉴权权限体系升级（补强原有鉴权能力）
1. 双层路由守卫防护：Next Middleware服务端前置守卫 + 客户端路由守卫，双层拦截无权限访问
2. 三维细粒度权限控制
  - 路由级：后端返回权限动态生成侧边栏菜单
  - 页面级：无权限直接跳转403页面
  - 按钮级：封装权限Hook，无权限隐藏/禁用操作按钮
3. 安全登录态存储：HttpOnly Cookie存Token + 客户端状态存用户信息，防范XSS攻击窃取令牌
2.4 React 组件与业务工程化
1. 业务逻辑抽离：封装通用业务Hook（分页、搜索、列表、表单），组件纯UI渲染，逻辑与视图解耦
2. React渲染性能优化：精准使用useMemo/useCallback、长列表虚拟滚动、组件合理拆分
3. 强类型表单方案：React Hook Form + Zod，实现同步/异步校验、表单联动、草稿缓存
4. 全局状态管理：基于Zustand做全局客户端状态模块化拆分，拆分用户信息、系统主题、全局配置三大模块，搭配状态持久化能力，页面刷新后全局配置不丢失
2.5 前端线上稳定性监控
1. 分层全局错误捕获：ErrorBoundary组件级报错、Next全局页面报错、全局运行时JS异常捕获
2. 简易前端监控：模拟Sentry实现错误日志上报、页面埋点、白屏检测、资源异常监控

4.3 微前端避坑说明
- 海外真实现状：海外团队更喜欢「单一清晰Next/React单体应用」，工程简洁、边界清晰、维护成本低；极少有老项目多框架并存、需要乾坤/qiankun模块拆分的场景
- 加了反而扣分：你的定位是偏前端全栈、深耕Next+React工程化，强行加微前端，面试官会认为：你追逐国内冗余技术、分不清业务场景、项目架构过度设计
- 面试回答话术（背下来即可）：I am familiar with micro-frontend concepts, but I choose not to introduce it in this demo. Because modern Next.js App Router + modular code splitting already meets large-scale project splitting needs, and micro-frontend will bring unnecessary runtime overhead and communication complexity. I prefer lean, maintainable production architecture.
- 替代最优方案（你正在做的方案）：采用独立 NPM component library + 业务项目分层，实现基建与业务解耦，兼顾复用性和简洁架构，远比微前端更适合现代Next企业级应用
